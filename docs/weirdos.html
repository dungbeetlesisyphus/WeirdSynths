CUSTOM LINUX DISTRO

WeirdOS

v 0 . 1 . 0

A custom Linux distribution that boots a Raspberry Pi 5 directly into VCV Rack with the WeirdSynths plugin pre-installed and face tracking running automatically. Power on, point a camera at your face, and patch.

System

Boot sequence

WeirdOS boots in approximately 30 seconds from power-on to playable patch.

0sPower on — kernel loads
aarch64 Linux 6.x on Cortex-A76, VideoCore VII GPU initialized
8sJACK audio server starts
48kHz, 256 frame buffer (~5ms latency), ALSA backend
12sCamera detected, bridge launches
MediaPipe face mesh initialized, UDP broadcast begins on ports 9000-9002
16sWeston compositor starts
Kiosk mode — full screen, no taskbar, no window decorations
22sVCV Rack launches
Default patch loads with NERVE, SKULL, MIRROR, VOICE pre-connected
~30sReady to patch
MIRROR display shows your face. NERVE outputs are live.
Build

Build the image

WeirdOS is built with Buildroot. You need a Linux x86_64 host with ~20GB free.

1

Install dependencies

Ubuntu/Debian build host required.

sudo apt install build-essential wget bc python3 \
  cpio unzip rsync libncurses-dev
2

Clone & build

The build script downloads Buildroot, configures it, and cross-compiles everything.

git clone https://github.com/dungbeetlesisyphus/WeirdSynths
cd WeirdSynths/weirdos
make
# First build: 1-3 hours. Subsequent builds: incremental.
3

Flash to SD card

make flash DEVICE=/dev/sdX
# Find your SD card: diskutil list (macOS) or lsblk (Linux)
4

First boot setup

On first boot, connect Ethernet and SSH in to complete setup:

ssh weird@weirdos.local   # password: weird
weirdos-setup              # configure camera, audio, install MediaPipe
CLI

weirdos command

Manage your appliance over SSH.

weirdos statusShow status of all services (VCV Rack, bridge, JACK, Weston)
weirdos startLaunch everything
weirdos stopStop all services
weirdos restartFull restart (useful after config changes)
weirdos configEdit system configuration in nano
weirdos log [name]View logs: jack, bridge, vcvrack, weston
weirdos cameraShow camera device info and capabilities
weirdos audioShow ALSA devices and JACK port list
weirdos-setupRe-run the first-boot setup wizard
Configuration

weirdos.conf

Edit /etc/weirdos/weirdos.conf (or run weirdos config):

CAMERA_DEVICE="/dev/video0"    # webcam device path
BRIDGE_PORT=9000               # UDP port for face tracking data

AUDIO_DEVICE="hw:0"            # ALSA device for JACK
JACK_SAMPLERATE=48000          # 44100 or 48000
JACK_BUFSIZE=256               # 64/128/256/512 — lower = less latency

MDNS_HOSTNAME="weirdos"        # accessible as weirdos.local
SSH_ENABLED=true               # remote access
CPU_GOVERNOR="performance"     # max CPU clock

After editing, run weirdos restart to apply changes.
